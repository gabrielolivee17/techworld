História do Java

Java foi criado pela equipe da Sun Microsystems, liderada por James Gosling, em 1995. A ideia era desenvolver uma linguagem orientada a objetos, segura, multiplataforma e com bom desempenho para dispositivos embarcados, como TVs inteligentes e aparelhos eletrônicos.

O lema "write once, run anywhere" (escreva uma vez, rode em qualquer lugar) definiu a proposta da linguagem: portabilidade. O código Java é compilado para bytecode, que é executado por uma máquina virtual (JVM), permitindo que o mesmo programa rode em qualquer sistema operacional com suporte à JVM.

Com o tempo, Java se tornou uma das linguagens mais populares do mundo, sendo amplamente usada em aplicações corporativas, sistemas bancários, Android, back-end web e muito mais. Em 2010, a Oracle adquiriu a Sun Microsystems e passou a manter o desenvolvimento do Java.

Sintaxe do Java

A sintaxe do Java é fortemente tipada, baseada em blocos e muito estruturada. Um programa Java precisa estar dentro de uma classe, e o ponto de entrada é sempre o método main().

  • Declaração de variáveis: precisa informar o tipo (ex: int, String, boolean)
  • Classes e métodos: estruturas essenciais do paradigma orientado a objetos
  • Modificadores: public, private, static, final
  • Compilação: Código Java (.java) é compilado em bytecode (.class) e executado pela JVM

Conceitos Fundamentais

O Java é uma linguagem robusta, orientada a objetos e fortemente tipada. A seguir, vamos explorar cada um dos seus conceitos-chave com exemplos práticos e explicações didáticas.

1. JVM (Java Virtual Machine)

A JVM é a máquina virtual que executa os programas Java. Ela permite que um programa Java seja compilado uma vez e executado em qualquer sistema operacional que tenha uma JVM instalada.

Como funciona: O código-fonte (.java) é compilado em bytecode (.class), que é interpretado pela JVM em tempo de execução.

2. POO - Programação Orientada a Objetos

Java é 100% orientado a objetos. Isso significa que tudo é feito a partir de classes e objetos. Os quatro pilares da POO são:

  • Encapsulamento: protege os dados da classe usando modificadores de acesso (private, public)
  • Herança: permite que uma classe herde propriedades e comportamentos de outra
  • Polimorfismo: objetos podem assumir muitas formas (ex: sobrescrita de métodos)
  • Abstração: expõe apenas o essencial ao usuário
class Animal { void fazerSom() { System.out.println("Som genérico"); } } class Cachorro extends Animal { void fazerSom() { System.out.println("Latido"); } } Animal a = new Cachorro(); a.fazerSom(); // Saída: Latido

3. Tipos Primitivos e Referência

Em Java, temos dois grupos principais de tipos de dados:

  • Primitivos: int, double, boolean, char, etc.
  • Referência: Objetos, como String, arrays e instâncias de classes
int idade = 25; // tipo primitivo String nome = "Carlos"; // tipo de referência (objeto)

4. Modificadores de Acesso

Controlam quem pode acessar membros (variáveis ou métodos) de uma classe.

  • public: acessível de qualquer lugar
  • private: acessível apenas dentro da própria classe
  • protected: acessível dentro do mesmo pacote ou por herança
  • sem modificador (package-private): acessível apenas no mesmo pacote

5. Compilação e Execução

O ciclo de vida de um programa Java:

  1. Você escreve o código em um arquivo .java
  2. Compila com javac MeuPrograma.java para gerar um .class
  3. Executa com java MeuPrograma

Esse processo garante portabilidade e segurança no ecossistema Java.

6. Tratamento de Exceções

Em Java, erros em tempo de execução são tratados com blocos try-catch. Isso evita falhas inesperadas no programa.

try { int resultado = 10 / 0; } catch (ArithmeticException e) { System.out.println("Erro: divisão por zero!"); } finally { System.out.println("Finalizando o bloco."); }

O bloco finally sempre será executado, mesmo se houver erro.

Exemplos de Código Java

1. Hello World

O exemplo mais básico em Java — imprimir uma mensagem na tela:

public class Main { public static void main(String[] args) { System.out.println("Olá, mundo!"); } }

2. Variáveis e tipos

Declaração de variáveis com tipo explícito:

int idade = 20; String nome = "Lucas"; double altura = 1.75; System.out.println(nome + " tem " + idade + " anos e " + altura + "m de altura.");

3. Condicionais

Decisões baseadas em condições lógicas:

int nota = 7; if (nota >= 6) { System.out.println("Aprovado!"); } else { System.out.println("Reprovado."); }

4. Laços de repetição

Repetições com for:

for (int i = 1; i <= 5; i++) { System.out.println("Passo " + i); }

5. Orientação a Objetos

Criação de uma classe com método:

public class Pessoa { String nome; Pessoa(String nome) { this.nome = nome; } void falar() { System.out.println("Olá, eu sou " + nome); } } Pessoa p = new Pessoa("Joana"); p.falar();

6. Tratamento de Exceções

Evita que o programa trave em caso de erro:

try { int resultado = 10 / 0; } catch (ArithmeticException e) { System.out.println("Erro: divisão por zero!"); }

Roadmap para Aprender Java

Java é uma linguagem poderosa e versátil usada em diversas áreas: aplicações corporativas, back-end, mobile (Android) e sistemas de larga escala. Veja abaixo um guia completo para dominar Java:

  1. Fundamentos: Sintaxe, variáveis, operadores, estruturas de controle, funções e métodos
  2. POO (Programação Orientada a Objetos): Classes, objetos, herança, encapsulamento, polimorfismo
  3. Tratamento de exceções: Blocos try/catch, throws, criação de exceções personalizadas
  4. Coleções: Listas, Conjuntos, Mapas (ArrayList, HashSet, HashMap, etc.)
  5. Manipulação de arquivos: Input/Output Streams, leitura e escrita com File, BufferedReader
  6. Interfaces gráficas: Swing e JavaFX
  7. Threads e Concorrência: Programação paralela com Runnable, Thread, ExecutorService
  8. Banco de Dados: Conexão com MySQL/PostgreSQL usando JDBC
  9. APIs modernas: Lambda Expressions, Streams, Optional, Date/Time API
  10. Desenvolvimento Web: Servlets, JSP, Spring Boot e REST APIs

Tópicos Avançados

Dominar Java também envolve aprender conceitos avançados e frameworks modernos:

  • Spring Boot: Criação rápida de APIs RESTful e aplicações robustas com segurança integrada
  • Hibernate e JPA: Mapeamento objeto-relacional para persistência de dados
  • Microserviços: Arquitetura distribuída com Spring Cloud, Docker e Kubernetes
  • Testes Automatizados: JUnit, Mockito, TestNG
  • Segurança: Autenticação e autorização com Spring Security e JWT
  • Design Patterns: Singleton, Factory, Observer, Strategy, Builder, Adapter
  • Desenvolvimento Android: Java como linguagem nativa para apps móveis

Conclusão

Java continua sendo uma das linguagens mais utilizadas no mundo corporativo. Sua performance, escalabilidade e comunidade forte fazem dela uma excelente escolha para quem busca estabilidade e empregabilidade no mercado de software.

Continue estudando, aplicando os conceitos em projetos reais e explorando novas tecnologias como Spring, Kafka, Docker e testes unitários.

Seja para criar apps Android, sistemas bancários ou servidores web, Java estará presente em muitos desafios do seu futuro como desenvolvedor.
